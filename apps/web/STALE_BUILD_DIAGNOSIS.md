## WHY THIS HAPPENED

1. **Old Build Process**: The app was previously built with a hardcoded `NEXT_PUBLIC_API_URL="https://service-pilot-b3df0d.appgen.co"`
2. **Static Export Created**: A static export was generated to `/out/` for Capacitor
3. **Android Build Cached**: The `/out/` was copied into `/android/app/src/main/assets/public/`
4. **Source Code Fixed**: The source files were updated to use `/api/...` relative paths
5. **But Stale Assets Remain**: The old compiled files in `/out/` and `/android/` still contain the hardcoded URL
6. **Dev Server Issue**: The dev server may still be serving from the stale `out/` directory in some cases

## IMPACT ON RUNTIME

### When Request is Made to `/api/user/by-email`:

**Old Compiled Code (from `out/`):**
```
Browser → POST https://service-pilot-b3df0d.appgen.co/api/user/by-email
         ↓
         Cross-origin request detected
         ↓
         Browsers default: Do NOT send cookies to different origins
         ↓
         Server receives request WITHOUT auth.session cookie
         ↓
         Backend: getSessionUserFromRequest() → null → Can't verify role
         ↓
         Response: 401 Unauthorized ❌
```

**Fresh Compiled Code (from `.next/`):**
```
Browser → GET /api/user/by-email
         ↓
         Same-origin request
         ↓
         Browser automatically includes all cookies (auth.session)
         ↓
         Server receives request WITH auth.session cookie
         ↓
         Backend: getSessionUserFromRequest() → { user_id, role } → ✅ Verified
         ↓
         Response: 200 OK with user data ✅
```
